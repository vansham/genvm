# ADR-010: GenVM versions

## Status

- **Status**: Accepted
- **Date**: 2025-08-06

## Context

GenVM may need to introduce changes

## Decision

Have multiple GenVM binaries

## Consequences

### Positive

- **Stability**: syncing won't have any issues
- **Variability**: contracts can request specific version

### Negative

- **Memory Consumption**: Each version is a few hundreds of MB
- **Modules**: It is not obvious how to handle many module versions

## Alternatives Considered

### If All Versions

This solution is bug prone

## Implementation Notes
